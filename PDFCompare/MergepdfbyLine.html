Example : Runtime Break And Merge PDF By Line as a Parameter

Where We Can Implement This Logic?

We are generating PDF form for all the stated in USA , may be for any Banking or Insurance document. Now for testing purpose we need 4-5 pages of a perticular section , not the entire PDF. And these section can very based on states. Now every time we are downloading the PDF and manually have to Scroll where my PDF is starting for each states. To avoid that , if we know 'Heading' / 'Any Initial Line of the perticular section' , then runtime our scripts can find that PDF then then start trimming it form that page.

Let's take an example of a PDF file which has 30 pages. Now if we split it , we will create 30 PDF files , where runtime after split we want to Merge pages using our string which would be the first page of the newly merged PDF. How to do that???

Below example is showing :
We have a PDF with 5 pages.
Splitting the PDF with All 5 Pages.
Passsing a runtime parameter as "THIS IS DUMMI PDF PAGE 3",1
This is nothing but creating a new merged PDF where my starting PDF has Sentence which is my input string as parameter.
Here '1' is my end page. So total two pages of document will be generated.
If we give "Any Line,5" .... then 6 pages of newly merged document will be generated. Make Sure the Line as a String parameter which we are passing must be Unique. And we must know length of our required PDF section.
Full end to end flow is happening runtime.

See below , one of my PDF has sentense as "THIS IS DUMMI PDF PAGE 3". Now we are adding one more parameter means end page. So if this sentence is found in page number '3' then it will create a new PDF with page number 3,4. In Example "Line Text" , 1 means starting page + 1.
