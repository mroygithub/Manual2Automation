Data Driven Testing Using Robot Framework

How to make test data as parameterized instead of hard coded

Robot : Data Driven Testing from .robot file.

Consider The Below Code ...

*** Settings ***
   Documentation   Data Driven Testing From .robot file
   Library     SeleniumLibrary
   Suite Teardown  Suite shutdown
   *** Variables ***
   
   
   *** Test Cases ***
   Data Driven Testing From .robot file
   
       Open Browser    https://techedtrainings.com/    Chrome
       Maximize Browser Window
       Wait For Condition  return document.title == "TechEd Trainings – IT Corporate Trainings"
       ${count} =  Get Element Count   xpath://h4[@class='infobox_title']
       Should Be True  ${count}==10
   *** Keywords ***
   Suite shutdown
       Close All Browsers
 

In above example it is given --
https://techedtrainings.com/ This is Test URL which can be stored under ***Variables*** section and call it from *** Test Cases *** section
Chrome This is Test Browser Name which can be stored under ***Variables*** section and call it from *** Test Cases *** section
"TechEd Trainings – IT Corporate Trainings" This is Page Title which can be stored under ***Variables*** section and call it from *** Test Cases *** section

So modified code can be written as below... 

*** Settings ***
   Documentation   Data Driven Testing From .robot file
   Library     SeleniumLibrary
   Suite Teardown  Suite shutdown
   *** Variables ***
   ${Browser}  Chrome
   ${url}  https://techedtrainings.com/
   ${pageTitle}    "TechEd Trainings – IT Corporate Trainings"
   
   *** Test Cases ***
   Data Driven Testing From .robot file
   
       Open Browser    ${url}     ${Browser}
       Maximize Browser Window
       Wait For Condition  return document.title == ${pageTitle}
       ${count} =  Get Element Count   xpath://h4[@class='infobox_title']
       Should Be True  ${count}==10
   *** Keywords ***
   Suite shutdown
       Close All Browsers

In above example it is given --
${Browser} Variable name of 'Chrome' browser
${url} Variable name of 'https://techedtrainings.com/' Test URL.
${pageTitle} Variable name of "TechEd Trainings – IT Corporate Trainings" Page Title.

Note:Now test URL , Browser Name , Page Title are not herd coded and we can call these variables from multiple tests as below... 

*** Settings ***
      Documentation   Data Driven Testing From .robot file
      Library     SeleniumLibrary
      Suite Teardown  Suite shutdown
      *** Variables ***
      ${Browser}  Chrome
      ${url}  https://techedtrainings.com/
      ${pageTitle}    "TechEd Trainings – IT Corporate Trainings"
      
      *** Test Cases ***
      TestCase_1
          Open Browser    ${url}     ${Browser}
          Maximize Browser Window
          Wait For Condition  return document.title == ${pageTitle}
          ${count} =  Get Element Count   xpath://h4[@class='infobox_title']
          Should Be True  ${count}==10
      
      TestCase_2
          Open Browser    ${url}     ${Browser}
          Maximize Browser Window
          Wait For Condition  return document.title == ${pageTitle}
          # DO SOME VALIDATION
      
      TestCase_3
          Open Browser    ${url}     ${Browser}
          Maximize Browser Window
          Wait For Condition  return document.title == ${pageTitle}
         # DO SOME VALIDATIONS
      
      
      *** Keywords ***
      Suite shutdown
          Close All Browsers

Robot : Data Driven Testing from XML file

Consider The Below XML File To Read

<testdata>
   <applicationurl>https://techedtrainings.com/ </applicationurl>
   <useid>testUser</useid>
   <password>Password$$#1</password>
   <chrome_path>/Users/mkroy/Downloads/chromedriver</chrome_path>
   <email>mithunroy.ee@gmail.com</email>
</testdata>
 

To make 'applicationurl' as data driven, we can write below code. How to read XML file this concept is already written in our previous acticle page 

*** Settings ***
               Documentation   XML Data Driven Testing
               Library     SeleniumLibrary
               Library     ../Library/readXML.py
               Suite Teardown  Suite shutdown
               *** Variables ***
               
               ${Browser}  Chrome
               ${pageTitle}    "TechEd Trainings – IT Corporate Trainings"
               
               *** Test Cases ***
               XML Data Driven Testing
                   ${url}=   readXML.read_xml   applicationurl
                   Open Browser    ${url}     ${Browser}
                   Maximize Browser Window
                   Wait For Condition  return document.title == ${pageTitle}
                   ${count} =  Get Element Count   xpath://h4[@class='infobox_title']
                   Should Be True  ${count}==10
               
               *** Keywords ***
               Suite shutdown
                   Close All Browsers

In above example it is given --
Library ../Library/readXML.py To import readXML.py python file.
${url} variable to capture applicationurl
readXML.read_xml Calling read_xml method present in readXML python File.
applicationurl XML node name as Parameter value
