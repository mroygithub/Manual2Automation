Run TestNG Using POM.XML

How To Run TestNG.XML By Maven POM.XML


First we need to set up Maven in our system :


Windows: Download maven from Here
Unzip it, set MAVEN_HOME as 'maven unzip path/bin'. Same in windows PATH as well.
MAC:Install maven from Terminal : brew install maven

Using Terminal check maven version as below:

mvn -v

Add below code in your pom.xml

  
  <properties>
         <maven.compiler.source>1.7</maven.compiler.source>
         <maven.compiler.target>1.7</maven.compiler.target>
  </properties>

After adding above code our basic POM.XML is look like :

    
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>con.IntelliJ.TestNG</groupId>
    <artifactId>con.IntelliJ.TestNG</artifactId>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>1.7</maven.compiler.source>
        <maven.compiler.target>1.7</maven.compiler.target>
    </properties>


    <dependencies>

        <dependency>
            <groupId>org.testng</groupId>
            <artifactId>testng</artifactId>
            <version>6.9.8</version>
            <scope>test</scope>
        </dependency>
        <!-- https://mvnrepository.com/artifact/org.seleniumhq.selenium/selenium-java -->
        <dependency>
            <groupId>org.seleniumhq.selenium</groupId>
            <artifactId>selenium-java</artifactId>
            <version>3.141.59</version>
        </dependency>
        <dependency>
            <groupId>org.apache.directory.studio</groupId>
            <artifactId>org.apache.commons.io</artifactId>
            <version>2.4</version>
        </dependency>

    </dependencies>

</project>


Our TestNG.XML is look like :

    
    <!DOCTYPE suite SYSTEM "https://testng.org/testng-1.0.dtd" >
    
    <suite name="manual2Automation Test Suite" parallel="false" thread-count="1">
    
    
        <listeners>
            <listener class-name="listeners.ItestListeners"/>
        </listeners>
    
        <test name="manual2Automation1" >
    
            <classes>
                <class name="TestNGPKG.testNGnew"/>
            </classes>
        </test>
    
    </suite>


Now we need to call above TestNG.xml from POM.xml

Adding 'maven-surefire-plugin' code as below: 

        <build>
            <plugins>
        
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-surefire-plugin</artifactId>
                    <version>3.0.0-M4</version>
                    <configuration>
                        <suiteXmlFiles>
                            <suiteXmlFile>src/pathToYour/TestNG.xml</suiteXmlFile>
                        </suiteXmlFiles>
                    </configuration>
                </plugin>
        
            </plugins>
        
        
        </build>


Now my complete POM.xml looks like below:

    
    <?xml version="1.0" encoding="UTF-8"?>
    <project xmlns="http://maven.apache.org/POM/4.0.0"
             xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
             xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
        <modelVersion>4.0.0</modelVersion>
    
        <groupId>con.IntelliJ.TestNG</groupId>
        <artifactId>con.IntelliJ.TestNG</artifactId>
        <version>1.0-SNAPSHOT</version>
    
        <properties>
            <maven.compiler.source>1.7</maven.compiler.source>
            <maven.compiler.target>1.7</maven.compiler.target>
        </properties>
    
        <build>
            <plugins>
    
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-surefire-plugin</artifactId>
                    <version>3.0.0-M4</version>
                    <configuration>
                        <suiteXmlFiles>
                            <suiteXmlFile>src/main/resources/myTestNG.xml</suiteXmlFile>
                        </suiteXmlFiles>
                    </configuration>
                </plugin>
    
            </plugins>
        </build>
    
    
        <dependencies>
    
            <dependency>
                <groupId>org.testng</groupId>
                <artifactId>testng</artifactId>
                <version>6.9.8</version>
                <scope>test</scope>
            </dependency>
            <!-- https://mvnrepository.com/artifact/org.seleniumhq.selenium/selenium-java -->
            <dependency>
                <groupId>org.seleniumhq.selenium</groupId>
                <artifactId>selenium-java</artifactId>
                <version>3.141.59</version>
            </dependency>
            <dependency>
                <groupId>org.apache.directory.studio</groupId>
                <artifactId>org.apache.commons.io</artifactId>
                <version>2.4</version>
            </dependency>
    
        </dependencies>
    
    </project>


Open Terminal or cmd --> go to your project path pom.xml file-->type 'mvn test'
This will start your execution and Terminal/cmd output will be as below:

This will execute all test inside 'myTestNG.xml' file

                    </suiteXmlFiles>
                </configuration>
            </plugin>
    
        </plugins>
    </build>


How To Pass TestNG.XML As A Parameter Through POM.xml?



This time in my POM.XML file, TestNG.xml can't be hard coded. So it can be written as:

    
    <build>
        <plugins>
    
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <version>3.0.0-M4</version>
                <configuration>
                    <suiteXmlFiles>
                        <suiteXmlFile>${suiteXmlFile}</suiteXmlFile>
                    </suiteXmlFiles>
                </configuration>
            </plugin>
    
        </plugins>
    </build>

Open Terminal or cmd --> go to your project path pom.xml file-->type 'mvn clean test -Dsurefire.suiteXmlFiles=src/main/resources/myTestNG.xml'
This will start your execution.



Pass Application URL As A Parameter Through POM.xml?



Add below line in TestNG.XML
applicationurl = parameter name
AppUrlValue = parameter value 

        
        <parameter name="applicationurl" value="${AppUrlValue}"></parameter>

Complete TestNG.XML can be written as:

    <!DOCTYPE suite SYSTEM "https://testng.org/testng-1.0.dtd" >
    
    <suite name="manual2Automation Test Suite" parallel="false" thread-count="1">
    
    
        <listeners>
            <listener class-name="listeners.ItestListeners"/>
        </listeners>
    
        <test name="manual2Automation1" >
            <parameter name="applicationurl" value="${AppUrlValue}"></parameter>
            <classes>
                <class name="TestNGPKG.testNGnew"/>
            </classes>
        </test>
    
    </suite>


My testclass can be written as:

    
    package TestNGPKG;
    
    import org.openqa.selenium.By;
    import org.openqa.selenium.WebDriver;
    import org.openqa.selenium.chrome.ChromeDriver;
    import org.testng.Assert;
    import org.testng.annotations.Parameters;
    import org.testng.annotations.Test;
    
    public class testNGnew {
    
        WebDriver driver;
    
    
        @Test
        @Parameters({"applicationurl"})
        public void testcase_1(String appurl)
    
        {
            System. setProperty("webdriver.chrome.driver", "/Users/chpath/Downloads/chromedriver2");
            driver = new ChromeDriver();
            driver.get(appurl);
            Assert.assertTrue(driver.findElement(By.id("hplogo")).isDisplayed());
            driver.quit();
        }
        
    }

Now Build section under POM.xml can be written as:

    <build>
        <plugins>
    
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <version>3.0.0-M4</version>
                <configuration>
                    <systemPropertyVariables>
                        <AppUrlValue>${AppUrlValue}}</AppUrlValue>
                    </systemPropertyVariables>
                    <suiteXmlFiles>
                        <suiteXmlFile>${suiteXmlFile}</suiteXmlFile>
                    </suiteXmlFiles>
                </configuration>
            </plugin>
    
        </plugins>
    </build>

Open Terminal or cmd --> go to your project path pom.xml file-->type 'mvn clean test -DsuiteXmlFile=src/main/resources/myTestNG.xml -DAppUrlValue=https://www.google.com'
This will start your execution.



Pass Multiple Parameters Through POM.xml?



Below Example Shows passing 4 parameters along with TestNG.xml
4 parameters are:
URL
Browser name
Release name
Environment Details

POM.xml build section can be written as :

    
    <build>
       
     <plugins>
    
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <version>3.0.0-M4</version>
                <configuration>
                    <systemPropertyVariables>
                        <AppUrlValue>${AppUrlValue}}</AppUrlValue>
                        <BrowserName>${BrowserName}}</BrowserName>
                        <ReleaseName>${ReleaseName}}</ReleaseName>
                        <EnvironmentName>${EnvironmentName}}</EnvironmentName>
                    </systemPropertyVariables>
                    <suiteXmlFiles>
                        <suiteXmlFile>${suiteXmlFile}</suiteXmlFile>
                    </suiteXmlFiles>
                </configuration>
            </plugin>
    
        </plugins>
    </build>

TestNG.xml section can be written as :

        <!DOCTYPE suite SYSTEM "https://testng.org/testng-1.0.dtd" >
        
        <suite name="manual2Automation Test Suite" parallel="false" thread-count="1">
        
        
            <test name="manual2Automation1" >
                <parameter name="applicationurl" value="${AppUrlValue}"></parameter>
                <parameter name="browsername" value="${BrowserName}"></parameter>
                <parameter name="releasename" value="${ReleaseName}"></parameter>
                <parameter name="environmentname" value="${EnvironmentName}"></parameter>
                <classes>
                    <class name="TestNGPKG.testNGnew"/>
                </classes>
            </test>
        
        </suite>
